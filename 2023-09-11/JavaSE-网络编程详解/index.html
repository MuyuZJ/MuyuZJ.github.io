概述

什么是网络

网络(Internet) 把分布在不同地理区域的计算机与专门的外部设备用通信线路互连成一个规模大,功能强的网络系统, 从而使众多的计算机可以方便地互相传递信息, 共享硬件 软件 数据信息等资源

偌大的地球变成了地球村

什么是网络编程

Java从语言级上提供了对网络应用程序的支持

Java实现了一个跨平台的网络库 实现了一个跨平台的网络库 隐藏底层细节 开发面向统一的网络编程环境

网络编程是针对网络连接的编程 而非网页编程(Web开发)

网络编程目的

直接或间接地通过网络协议与其他计算机实现数据交换及进行通讯

网络通信要素

  • IP地址和对应端口
  • 通信协议

类比于填写信件 需要指定本机的地址和对方地址

网络通信也需要指定 本机地址和对方地址 (IP和对应的端口) 使用一定的协议进行连接后 可以进行通信

对应需要解决的问题:

  • 如何正确定位到网络中的具体的一台或者多台主机(IP地址) 及对应的应用(端口)
  • 如何进行可靠高效的数据连接

网络参考模型

网络通信协议有两套参考模型:

  • OSI参考模型: 模型过于理想化 未能广泛推广
  • TCP/IP五层协议: 实际使用的模型 实际的国际标准

IP地址

IP地址按照版本分可以分为:

  • IPv4: 由4字节组成 以点分十进制表示 (192.168.1.10)
  • IPv6: 由16字节组成 写成8个使用四个十六进制位表示的无符号整数 冒号分隔 (2001:0db8:3c4d:0015:0000:0000:1a2f:1a2b)

IP地址按照访问权限分为:

  • 公网IP地址: 在互联网(公网)上可以被访问到
  • 私有IP地址: 仅限于局域网使用 公网访问不到
    • 常见的192.168.xx.xx 即私有地址 专门为内部使用

IP地址中也有特殊的地址:

  • localhost 或 127.0.0.1 表示本地回环地址(hostAddress) 表示本机的IP地址

域名

IP (以IPv4为例) 以四个0~255的数字构成 不方便记忆 故有了域名

史上第一个域名(不排除时间被重置可能) Nordu.net

域名是由一串用点分隔的名字组成的互联网上某一台计算机或计算机组的名称 (http://www.baidu.com/ )

域名解析

域名解析服务将域名解析成对应的IP地址返回 是因特网的意向核心服务

域名和IP是一一对应的 通过域名 我们可以定位到具体的某台资源主机

端口号

一个端口对应主机的一个程序 不同的进程有不同端口号 可以用来区分应用

端口号被规定为16位整数 即0~65535

TCP和UDP各有65535个端口 单协议下端口不能冲突

常用的端口

  • 公认端口 (0~1023)
    • HTTP: 80
    • HTTPS: 443
    • FTP: 21
    • Telnet: 23
  • 注册端口 (1024~49151)
    • Tomcat: 8080
    • MySQL: 3306
    • Oracle: 1521
  • 动态 私有端口 (49152~65535)
    • 可以用于自定义应用

可以通过命令行查看端口

网络套接字

网络套接字 (Socket) 是一个抽象层,应用程序可以通过它发送或接收数据,可对其进行像对文件一样的打开 读写和关闭等操作. 套接字允许应用程序将I/O插入到网络中, 并与网络中的其他应用程序进行通信. 简而言之, 网络套接字是IP地址与端口的组合

实际情况下 应用程序的通信就是Socket(IP+端口)通信 网络编程就是面向Socket编程

网络通信协议

网络通信协议即计算机通信的约定 对速率, 传输代码, 代码结构等进行规范

计算机网络通信设计内容繁多且复杂 需要指定源地址/目的地址 加密解密 压缩解压缩 差错控制 流量控制等

根据网络参考模型:

  • 网络通信过程可以分为多层
  • 每一层具有每一层对应的协议
  • 同层级之间可以通信
  • 上下层没有直接关系 不互通
  • 上层调用下层 下层为上层提供服务

其中 TCP和UDP是传输层常用协议 IP是网络层传输协议

TCP和UDP

TCP 即用户传输协议(Transmission Control Protocol) UDP 即用户数据报协议(User Datagram Protocol)

常言的TCP/IP其实是两个协议的组合: 传输层协议TCP 和 网络层协议IP

TCP/IP协议模型从更实用的角度出发 形成了高效的四层体系结构 即物理链路层,IP层,传输层和应用层

TCP和UDP区别

TCP UDP
连接情况 有连接 (三次握手) 点对点通信 无连接 点对多通信
传输包大小 可以进行大数据量传输 限制 <64K
是否可靠 可靠 不可靠
是否广播 否 是
传输资源消耗 较高 速度相对慢 开销小 速度快

UDP网络编程

UDP网络编程 基于DatagramSocket和DatagramPacket类

  • UDP 数据报通过数据报套接字 DatagramSocket 发送和接收 系统不保证UDP数据报一定能够安全送到目的地或什么时候可以抵达
  • DatagramPacket 对象封装了UDP数据报 在数据报中包含了发送端的IP地址和端口号以及接收端 的IP地址和端口号
  • UDP协议中每个数据报都给出了完整的地址信息 因此无需建立发送方和接收方的连接 如同发快递包裹一样

URL网络编程

什么是URL

URL 即Uniform Resource Locator, 统一资源定位符 是互联网上标准资源的地址

URL格式

URL常见格式为 scheme://host[:port#]/path/…/[;url-params][?query-string][#anchor]

其分别含义为

名称 含义
scheme 协议类型 [http / https / ftp / ed2k / thunder …]
host HTTP服务器的IP地址或域名
port HTTP服务器对应端口 默认为80 其他需要指明
path 资源访问路径
url-params url所带参数
query-string 发送给HTTP服务器的数据
anchor 锚点定位
