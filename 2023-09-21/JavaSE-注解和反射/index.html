注解

什么是注解

注解(Annotation) 是自JDK5.0引入的新技术

注解和注释:

注解有注释的功能 并自身有一定的功能

注释只是对程序代码的解释

注解作用:

  • 注解不是程序本身 可以对程序做出解释 (和注释类似)
  • 可以被其他程序(例如编译器)读取 有一定的功能

注解格式:

  • 注解样式如 @Xxxx 存在
  • 部分注解还可以添加参数

注解使用:

  • 注解可以附加在package / class / method / field等上面 添加额外的信息
  • 可以通过反射机制实现对这些元数据的访问

元数据

关于该文件或对象的一切信息都是元数据

例如: 技术相关的信息 或 内容信息

内置注解

内置注解 即JDK自带注解

知乎 - JDK内置注解

  • @Deprecated

    • 可以注释于 属性 / 类
    • 已废弃 (不推荐开发者使用)

  • @Override

    • 用于修饰方法
    • 声明该方法意欲重写超类的某方法声明

  • @SuppressWarning

    • 抑制编译时的警告信息

    • 需要添加参数 以下是举例

  • @FunctionalInterface

    • 指定接口必须为函数式接口 如果声明但是方法不是函数式接口 则会报错

  • @SafeVarargs

    • 用于抑制堆污染警告

Java 堆污染

Heap pollution(堆污染) 指的是当把一个不带泛型的对象赋值给一个带泛型的变量时 就有可能发生堆污染.

由于定义带泛型变量时并不强制指定泛型类型, 因此如果借此发生狸猫换太子的操作的话, 那么就会导致堆污染. 堆污染在编译时并不会报错, 只会在编译时提示有可能导致堆污染的警告. 在运行时,如果发生了堆污染, 那么就会抛出类型转换异常.

元注解

元注解的作用就是负责注解其他注解

JDK定义了4个标准的meta-annotation类型 可以在java.lang.annotation包中找到

  • @Target

    • 用于描述注解的使用范围 可以用于哪里

  • @Retention

    • 表示需要在什么级别保存该注释信息 描述注解的生命周期
    • SOURCE < CLASS < RUNTIME
      • Class 注释将由编译器记录在类文件中，但VM不需要在运行时保留
      • Runtime 注释将由编译器记录在类文件中，并由VM在运行时保留，因此可以反射读取
      • Source 注释将被编译器丢弃

  • @Documented

    • 说明该注解将被包含在javadoc中

  • @Inherited

    • 说明子类可以继承父类中的该注解

使用@Interface定义自定义注解 自动继承Annotation接口

@ interface用来声明一个注解 格式 public @ interface 注解名 { 定义内容 } 其中的每一个方法实际上是声明了一个配置参数.

方法的名称就是参数的名称. 返回值类型就是参数的类型 ( 返回值只能是基本类型,Class , String , enum ). 可以通过default来声明参数的默认值

如果只有一个参数成员 , 一般参数名为value

注解元素必须要有值 , 我们定义注解元素时 , 经常使用空字符串,0作为默认值 .

静态语言和动态语言

静态语言:

  • 运行时结构无法改变

动态语言

  • 运行时可以改变其结构的语言 可以根据代码运行的条件改变自身结构

Java不是动态语言 但又有一定的动态性 可以称之为”准动态语言”

开发者可以利用反射机制获得类似动态的语言特性 反射是Java被视为动态语言的关键

Java反射

Java中 可以通过如下方式 获得任意对象的类对象

这样就可以在堆内存方法中产生Class类型的对象

利用反射 可以动态执行:

  • 获得任意对象所属类
  • 构造任意一个类的对象
  • 判断任意一个类的成员变量和方法
  • 获取泛型信息
  • 调用任意对象的变量或方法
  • 处理注解
  • 生成动态代理
  • … …

可以获取到运行时类的完整结构 Field / Method / Constructor / Superclass / Interface / Annotation

实际开发不会经常使用 但是需要熟悉 java.lang.reflect包的作用和反射机制

Java反射的优缺点:

  • 优点
    • 可以实现动态创建和编译对象 体现Java一定的灵活性
  • 缺点
    • 对性能有一定影响 反射实质是解释操作 告诉JVM需要做什么 慢于直接执行相同操作

内存分析

Java对象存储 内存分析

Java加载类内存分析

当程序主动使用某类 但是类未被加载 则会执行如下步骤进行初始化

  • 类加载 Load
    • 加载文件字节码到内存中
    • 将静态数据转换成方法区的运行时数据结构
    • 生成Class对象 (java.lang.Class)
  • 类链接 Link
    • 验证 确保加载的类信息符合JVM规范 没有安全方面的问题
    • 准备 正式为类变量(static)分配内存并设置类变量默认初始值 这些内存都将在方法区中进行分配
    • 解析 虚拟机常量池内的符号引用(常量名)替换为直接引用(地址)的过程
  • 类初始化 Initialize
    • 执行类构造方法的过程 该方法是由编译期自动收集类中所有类变量的赋值动作 和静态代码块中的语句合并产生的
    • 当初始化一个类的时候 如果发现其父类还没有进行初始化 则需要先触发其父类的初始化
    • 虚拟机会保证一个类的方法在多线程环境中被正确加锁和同步

类初始化发生动机

  • 类主动引用
    • 虚拟机启动 会初始化main方法所在的类
    • new类生成类的对象
    • 调用类的静态成员(除了final常量)和静态方法
    • 使用反射(java.lang.reflect)进行反射调用
    • 初始化子类 (而父类没有初始化) 会初始化父类
  • 类被动引用
    • 当访问一个静态域时 只有真正声明这个域的类才会被初始化
    • 通过数组定义类引用 不会触发此类的初始化
    • 引用常量不会触发此类的初始化 (在链接阶段就存入调用类的常量池中)

类加载器

知乎 - 类加载器

Java语言是一种具有动态性的解释语言 类(CLASS) 只有被加载到 JVM 中后才能运行

类加载器就是加载字节码文件(.class)的类

JVM规范了如下类加载器

  • 引导类加载器 Bootstrap Classloader
  • 扩展类加载器 Extension Classloader
  • 系统类加载器 System Classloader

加载方式:

  • 隐式加载: 使用new关键字等方式创建对象
  • 显式加载: 使用Class.forName()加载类

标准的JavaSE类加载器具有缓存 当类被加载时 会放入缓存维持加载 JVM垃圾回收机制可以回收这些类对象

如果参数值为true则代表对象是用取消java语言访问检查 反之则事实访问检查

反射操作泛型

Java采用泛型擦除的机制引入泛型 Java的泛型仅用于编译器javac 免去强制转换问题 确保数据安全性

当编译完成 所有泛型类型都将被擦除

Java新增了几种类型来代表不能被归一到Class类中的类型但是又和原始类型齐名的类型

  • ParameterizedType : 表示一种参数化类型,比如Collection
  • GenericArrayType : 表示一种元素类型是参数化类型或者类型变量的数组类型
  • TypeVariable : 是各种类型变量的公共父接口
  • WildcardType : 代表一种通配符类型表达式
