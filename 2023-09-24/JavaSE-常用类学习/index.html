Shallow Clone 和 Deep Clone (浅克隆和深克隆)

Shallow Clone only copies the value type of a class. If a reference type is present in class, then the reference is copied, but the referred object is not copied. 关于Java垃圾回收

  • 垃圾回收通常只在内存空间濒临用完的时候
  • 垃圾回收不等于C++的析构
  • 垃圾回收和内存有关 用于回收程序不再使用的内存空间

finalize()方法不管对象如何创建(可能调用本地方法或者非Java代码生成) 垃圾回收期都会负责释放占据的所有内存

hashCode()

什么是哈希码

哈希码（Hash code）是根据输入数据而生成的固定长度的数字或字符串。它是将数据（如字符串、文件或对象）转换为相对唯一的、固定长度的值的算法的输出。哈希码通常用于数据的索引、加密、身份验证和数据匹配等领域。

一个好的哈希算法应该能够在输入数据发生微小变化时产生完全不同的哈希码，并且尽量避免不同的输入数据生成相同的哈希码（即冲突） 通常应具有均匀分布和高度随机性，以最大限度地减少冲突的可能性

哈希码是单向的，即从哈希码无法还原出原始输入数据。此外，不同的输入数据可能会产生相同的哈希码（冲突），但好的哈希算法应该将冲突的可能性降到最低。

该方法用于返回对象的哈希码值

该方法用于哈希查找 可以减少查找中使用equals()方法的次数 重写equals通常也需要重写hashCode方法

类中方法定义 它是一个公开的原生方法

根据哈希码介绍 可以发现 同一个对象的哈希码是一样的 不同的对象哈希码一般而言是不同的(除非特殊情况)

包装类

什么是包装类

基本类型数据不具备对象的特点 (无属性 无方法) 只能进行常规处理

Java是典型的面向对象的编程语言 为了简单有效地处理数据 Java为每种基本数据类型分别设计了对应包装类 称之为包装类

单精度实数型 float Float

布尔型 boolean Boolean

Java为包装类提供了一些有用的方法

包装类对象一经创建 其内容(封装的基本数据类型数据值)不可改变

装箱和拆箱

基本数据类型可以和包装类型相互转换:

  • 由基本数据类型转换为包装类型 称为装箱
  • 由包装类型转换为基本数据类型 称为拆箱

自动装箱和拆箱实质是编译器执行 根据编译时的语法决定是否进行装箱或拆箱

自动装箱和自动拆箱是所谓的语法糖 使用比较方便 但实际使用需要了解其本质运行原理

自动装箱和自动拆箱是常用的功能 需要重点掌握

String类

String类表示一个字符串

Java中所有字面量都是其实例 它是最常用的类之一

String是具体的类 它不是基本数据类型

String其实不是动态概念 String的实例对象是不可变的 它的值在创建后无法更改

String类有如下成员变量

String的不可变性

String使用频率高 为了”安全性”和”效率”

  • 若允许更改则可能被篡改
  • 若允许被继承则会降低程序性能 (需要考虑谁可以修改该对象)
  • 因为String作为HashMap HashTable等的key 不可更改可以大大提升执行效率
  • 避免重复创建的开销

CSDN - 深入理解String源码与底层

深入理解final关键字

  • 用于修饰类 成员变量和成员方法
  • final修饰的类所有的方法都不能被重写 但是可以被重载
  • final修饰的元素无法更改 除非再创建一个新的String对象

普通字符串和String类

内存分析

可以使用intern() 方法 将字面量手动入池

Java中 String类过于常用 故开发者为其采用享元模式 即字符串被添加到共享池 当后续再次生成同内容字符串实例 会直接共享此对象 不必再创建另一个字符串

两种方式区别

  • 直接赋值 String str = "Hello" 只会开辟一块堆内存空间 会自动入池 不会产生垃圾
  • 构造方法 String str = new String("Hello") 会开辟两块内存空间 其中一块会变成垃圾 而后会被系统回收 不能自动入池 需要使用intern() 方法手动入池
  • 开发过程通常不会采用构造方法进行字符串实例化

字符串常量池

通常字符串会经过如下几处:

  • 常量池表
    • 该表式Class文件中存储所有常量的Table 是Class文件中的内容 不是运行时内容 只是字节码指令
  • 运行时常量池
    • 为内存方法区的一部分 是运行时内容 绝大部分是Class常量池表转换而来 也包含了某些动态加入的内容
  • 字符串常量池
    • JVM实例全局共享 在方法区中 HotSpot设置哈希表StringTable 用于引用堆中字符串实例 被引用即驻留

三者比较

  • String
    • 字符串常量
    • 执行速度慢 每次改变相当于又创建了一个String对象 经常变化的对象建议不要使用String (可能GC介入)
  • StringBuffer
    • 字符串变量 线程安全 适合用于多线程情况下
    • 执行速度较快
  • StringBuilder
    • 字符串变量 非线程安全
    • 执行速度快(大部分情况下) 建议优先使用
