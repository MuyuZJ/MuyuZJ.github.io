涉及文件的读写 就需要使用到IO流

什么是IO流

Java程序中 所有的数据输入和输出都以流的方式进行 称之为流式传输

JDK提供了各种各样的流 以适应不同的需求

IO流即输入(Input)输出(Output)流 通过IO流可以完成文件的读写

IO流根据内存为参照物 按照流向进行分类

  • 磁盘往内存写入数据 称为读
  • 内存往磁盘输出数据 称为写

可以看到内存输出相当于磁盘输入 磁盘输出相当于内存输入

默认谈论输入输出流以内存(程序)角度

Java有用于读写数据的类 File 它的实例只是内存的一个对象 真正文件是磁盘某块空间

如何读取和写入 其实和饮水机结构相似

  • 读取
    • 扣上水桶(放入文件) 打开水龙头(开启输入流)
  • 写入
    • 接上水桶(准备文件) 打开水龙头(开启输出流)

如若输入输出的数据不规范还可以套上过滤器(处理流) 以达到需要的效果

本章节 着重讲解节点流 处理流笔墨相对少

InputStream / OutputStream / Reader / Writer 都是虚拟类 实例调用虚类的实现类展示

IO流分类

输入输出流定义在java.io包下

可以从不同角度进行划分

  • 按照数据流方向划分 可以分为输入流和输出流
  • 按照处理数据单位划分 可以分为字节流和字符流
  • 按照功能划分 可以分为节点流和处理流

字节流和字符流

  • 字节流处理数据是以0和1表示的最底层数据表示形式 按照字节(1Byte/8bit)读取
  • 字符流 按照字符(2字节) 依次读取

JDK提供有如下抽象流类型 可供使用

字节流 字符流
输入流 InputStream Reader
输出流 OutputStream Writer

通过流 可以实现程序和数据源之间的数据交换

后文会着重讲述四种抽象流类型

节点流和处理流

  • 节点流是程序直接读取或者写入数据源的数据 File是节点流的代表
  • 处理流是包在其他流上的流

缓冲流

缓冲流好比”套接”在相应节点流上 对读写数据提供缓冲 功能 提高读写效率

Java提供了如下接口 可以用于构建缓冲流

对于缓冲流

  • 支持对应的父类的mark和reset方法
    • 保证mark有效的情况下限制读取的字符数 当读取字符达到或超过此限制时 尝试重置流会失败 当限制数值大于输入buffer的默认大小时
      将会动态分配一个容量不小于限制数值的buffer 慎用大数值
    • 调用reset方法时 将回到被定位的位置 再次遍历
  • BufferedReader提供了readLine方法用于读取一行字符串 BufferedWriter提供了newLine用于写入一个行分隔符
  • 对于输出的缓冲流 写出的数据会现在内存中缓存 使用flush方法将会使内存中的数据立刻写出

缓冲流设置了一个桶 将缓存数据存储到该桶中 再写到磁盘 比起直接写入磁盘的方案 可以有效减少磁盘IO的次数 利于保护磁盘

数据流

DataInputStream 和 DataOutputStream 提供了可以存取与机器无关的Java原始类型数据(int, double等)的方法

DataInputStream 和 DataOutputStream 属于处理流 (套接在节点流上) 继承自InputStream和OutputStream

写入数据后再读取

InputStream 和 OutputStream 不会对内容进行判断 读取不会区分不同的数据类型

DataInputStream 和 DataOutputStream 会对内容进行判断 区分不同的数据类型
