概述

什么是GUI

GUI 即Graphics User Interface, 图形用户界面

GUI开发是Java开发的一部分 但并非强项 因为外观不太美观, 组件数量偏少且需要大量JRE环境 因而没有流行起来

为什么需要学习GUI编程

既然使用Java开发GUI有很多劣势 但是我们还是需要简单学习和使用

  • Java的GUI组件(JTable JList等) 很多都是MVC的经典示范 了解监听
  • 工作可能(虽然很小可能)会维护AWT/Swing开发的软件
  • 可以串联以往的知识 编写自己感兴趣的软件
  • 技多不压身

JavaGUI的核心技术: Swing, AWT

Swing建立在AWT技术上 故需要先学习AWT 再学习Swing

AWT

什么是AWT

AWT 即 Abstract Window Toolkit, 抽象窗口工具包 用于Java应用的GUI开发

其各种GUI元素(窗口 按钮 文本框等) 都使用Java进行实现

相关包都存在于java.awt包及其子包下

组件和容器

组件和容器都是AWT的核心类

  • 组件(Component) 代表所有可见的图形元素
  • 容器(Container) 为可以容纳其他Component元素的容器
    • Window可以作为单独的应用程序窗口展示
    • Panel则无法单独作为应用程序窗口展示 需要装入window中

Window分为Frame(一般窗口)和Dialog(弹出窗口, 对话框)

Panel曾在Applet(Java的页面小程序)中被广泛使用 后被Ajax和JavaScript取代

布局管理器

Java提供过了布局管理器对象 可以管理容器布局

AWT提供了五种布局管理器:

  • FlowLayout
    • Panel的默认布局管理器
    • 行内组件从左到右 一行排满后换行展示
    • 不改变原有组件尺寸 可以设置不同的间距 行距 对齐方式
  • BorderLayout
    • Frame的默认布局管理器
    • 将整个容器布局分为: 东 西 南 北 中 五个区域 (默认会被添加到Center区域)
    • 组件只能被添加到对应的区域
    • 每个区域只能加入一个组件 如果加入多个组件则会被覆盖
  • GridLayout
    • 将容器控件分割成规则的矩形区域 每个单元格区域大小相等
    • 组件被添加到每个单元格中 从左到右 自上而下
  • CardLayout
    • GridLayout 将容器切割为棋盘一样m行n列的网格 每个网格可以放置一个组件 添加到容器的组件从左向右自上而下依次放置
    • 放置的组件自动占据网格的整个区域,每个组件的大小相同 不能改变组件大小
  • GridBagLayout
    • 相当于升级版本的GridLayout 允许网格大小互不相同
