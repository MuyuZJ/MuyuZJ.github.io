多线程详解

线程

线程是指在一个程序中执行的一条独立的路径 是程序中的执行流 用于实现并发执行多个任务

每个线程都有自己的执行路径和资源 可以独立地执行任务

程序运行至少有一个线程 否则没有存在的意义

线程是操作系统调度的基本单位 可以并发执行 提高程序的效率和响应速度

程序 进程和线程

  • 程序是语言编写的指令集合 指一段静态的代码
  • 进程是动态概念 是指一个正在运行的程序 指一个程序的一次执行过程 具有生命周期
    • 进程状态: 就绪态 运行态 阻塞态
  • 进程中的一个单一顺序的控制流 作为调度和执行的最小单位 每个线程都拥有独立的运行栈和程序计数器 切换的开销小 共享同一个进程中的结构(方法区 堆)
    • 线程状态: New Runnable Blocked Waiting Timed-Waiting Terminated

多线程

多线程是指在一个程序中 同时运行多个线程 每个线程可以独立执行不同的任务

在多线程的情况下 程序可以并发执行多个任务 提高程序的效率和响应速度

多线程和多任务

多线程是指在一个程序中同时执行多个线程 每个线程可以独立执行不同的任务

多任务是指在操作系统层面同时执行多个任务 可以是多个进程或线程同时运行

多线程通常由单CPU模拟多CPU执行 真正的多线程需要多CPU

多线程中 线程互相独立 在各自工作内存中交互 合理控制对应资源

进程开辟线程后 由调度器安排 人为无法干预

线程调度需要一定的资源开销

线程实现

自JDK1.0开始 Java就支持Thread类

线程实现有三种方式

  • 继承Thread类
  • 实现Runnable接口
  • 实现Callable (了解)

查看JDK帮助文档 对Thread的解释

A thread is a thread of execution in a program. The Java Virtual Machine allows an application to have multiple threads of execution running concurrently.

翻译: 线程是程序中的执行线程。Java虚拟机允许应用程序同时运行多个执行线程。

Every thread has a priority. Threads with higher priority are executed in preference to threads with lower priority. Each thread may or may not also be marked as a daemon. When code running in some thread creates a new Thread object, the new thread has its priority initially set equal to the priority of the creating thread, and is a daemon thread if and only if the creating thread is a daemon.

翻译: 线程是程序中的执行线程。Java虚拟机允许应用程序同时运行多个执行线程。每个线程都有一个优先级。优先级高的线程将优先于优先级低的线程执行。每个线程可能也可以被标记为守护线程。当某个线程中的代码创建一个新的Thread对象时，新线程的优先级最初设置为创建线程的优先级，并且当且仅当创建线程是守护线程时，新线程才是守护线程。

When a Java Virtual Machine starts up, there is usually a single non-daemon thread (which typically calls the method named main of some designated class). The Java Virtual Machine continues to execute threads until either of the following occurs:

  • The exit method of class Runtime has been called and the security manager has permitted the exit operation to take place.
  • All threads that are not daemon threads have died, either by returning from the call to the run method or by throwing an exception that propagates beyond the run method.

翻译: 当Java虚拟机启动时，通常会有一个非守护线程（通常调用某个指定类的main方法）。Java虚拟机会继续执行线程，直到以下情况之一发生：

  1. Runtime类的exit方法已被调用，并且安全管理器已允许进行退出操作。
  2. 所有非守护线程已经终止，可能是通过从run方法返回，或者是通过抛出超出run方法的异常。

There are two ways to create a new thread of execution. One is to declare a class to be a subclass of Thread. This subclass should override the run method of class Thread. An instance of the subclass can then be allocated and started.

翻译: 创建新的执行线程有两种方法。一种方式是将一个类声明为Thread的子类。这个子类应该重写Thread类的run方法。然后可以创建并启动这个子类的实例。

The other way to create a thread is to declare a class that implements the Runnable interface. That class then implements the run method. An instance of the class can then be allocated, passed as an argument when creating Thread, and started.

翻译: 创建线程的另一种方式是声明一个实现了 Runnable 接口的类。然后，该类需要实现 run 方法。可以将该类的实例分配内存空间，并在创建 Thread 时将其作为参数传递，并启动线程。

Every thread has a name for identification purposes. More than one thread may have the same name. If a name is not specified when a thread is created, a new name is generated for it. Unless otherwise noted, passing a null argument to a constructor or method in this class will cause a NullPointerException to be thrown.

每个线程都有一个名称用于标识目的。多个线程可以有相同的名称。如果在线程创建时没有指定名称，系统会为其生成一个新的名称。除非另有说明，否则在此类中将null参数传递给构造函数或方法将导致抛出NullPointerException异常。

继承Thread类

  1. 自定义线程类 继承Thread类
  2. 重写run方法 编写线程执行体
  3. 创建线程对象 调用start方法启动线程

运行发现 main方法和run方法交替执行

实现Runnable接口

  1. 创建自定义线程类 实现Runnable接口
  2. 重写run方法 编写线程体
  3. 创建线程对象 传入实现类 调用start方法启动线程

运行程序 可以发现main函数与run函数交替执行

该方法采用了代理模式 一份资源 多个代理

为了避免OOP单继承局限性 建议使用实现Runnable接口的方式 更加灵活

  4. 创建执行服务
  5. 提交执行 获得结果
  6. 关闭服务
